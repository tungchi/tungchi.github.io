java基础(4)-java内存管理

java基础(4)-java内存管理

2.java内存分配研究:http://www.blogjava.net/Jack2007/archive/2008/05/21/202018.html

java程序运行过程

以下这段话引用自参考1

首先Java源代码文件(.java后缀)会被Java编译器编译为字节码文件(.class后缀)，然后由JVM中的类加载器加载各个类的字节码文件，加载完毕之后，交由JVM执行引擎执行。在整个程序执行过程中，JVM会用一段空间来存储程序执行期间需要用到的数据和相关信息，这段空间一般被称作为Runtime Data Area（运行时数据区），也就是我们常说的JVM内存。因此，在Java中我们常常说到的内存管理就是针对这段空间进行管理（如何分配和回收内存空间）。

运行时数据区

运行时数据区主要包括以下:
程序计数器(Program Counter Register)、Java栈(VM Stack)、本地方法栈(Native Method Stack)、方法区(Method Area)、堆(Heap)。

运行时数据区每部分的作用

程序计数器(Program Counter Register):寄存器,保存当前程序执行指令的地址,在JVM中，多线程是通过线程轮流切换来获得CPU执行时间的，因此，在任一具体时刻，一个CPU的内核只会执行一条线程中的指令，因此，为了能够使得每个线程都在线程切换后能够恢复在切换之前的程序执行位置，每个线程都需要有自己独立的程序计数器，并且不能互相被干扰，否则就会影响到程序的正常执行次序。因此，可以这么说，程序计数器是每个线程所私有的。

java栈:Java栈中存放的是一个个的栈帧，每个栈帧对应一个被调用的方法，在栈帧中包括局部变量表(Local Variables)、操作数栈(Operand Stack)、指向当前方法所属的类的运行时常量池（运行时常量池的概念在方法区部分会谈到）的引用(Reference to runtime constant pool)、方法返回地址(Return Address)和一些额外的附加信息。当线程执行一个方法时，就会随之创建一个对应的栈帧，并将建立的栈帧压栈。当方法执行完毕之后，便会将栈帧出栈。

本地方法栈:与java栈类似,不过本地方法栈是为native方法服务的.

堆:Java中的堆是用来存储对象本身的以及数组（当然，数组引用是存放在Java栈中的）。只不过和C语言中的不同，在Java中，程序员基本不用去关心空间释放的问题，Java的垃圾回收机制会自动进行处理。因此这部分空间也是Java垃圾收集器管理的主要区域。另外，堆是被所有线程共享的，在JVM中只有一个堆。

方法区:方法区在JVM中也是一个非常重要的区域，它与堆一样，是被线程共享的区域。在方法区中，存储了每个类的信息（包括类的名称、方法信息、字段信息）、静态变量、常量以及编译器编译后的代码等。

以上内容引用自参考1

总结

VM是基于堆栈的虚拟机.JVM为每个新创建的线程都分配一个堆栈.也就是说,对于一个Java程序来说，它的运行就是通过对堆栈的操作来完成的。堆栈以帧为单位保存线程的状态。JVM对堆栈只进行两种操作:以帧为单位的压栈和出栈操作。
我们知道,某个线程正在执行的方法称为此线程的当前方法.我们可能不知道,当前方法使用的帧称为当前帧。当线程激活一个Java方法,JVM就会在线程的Java堆栈里新压入一个帧。这个帧自然成为了当前帧.在此方法执行期间,这个帧将用来保存参数,局部变量,中间计算过程和其他数据.

classpath:下面一段话引用自参考2

注意：在定义classpath环境变量时，需要注意的情况
如果没有定义环境变量classpath，java启动jvm后，会在当前目录下查找要运行的类文件；
如果指定了classpath，那么会在指定的目录下查找要运行的类文件。
还会在当前目录找吗？两种情况：
1）：如果classpath的值结尾处有分号，在具体路径中没有找到运行的类，会默认在当前目录再找一次。
一般不指定分号，如果没有在指定目录下找到要运行的类文件，就报错，这样可以调试程序。
2）：如果classpath的值结果出没有分号，在具体的路径中没有找到运行的类，不会再当前目录找。

关键字,保留字

关键字（keyword）:Java的关键字对java的编译器有特殊的意义，他们用来表示一种数据类型，或者表示程序的结构等。
保留字(reserved word)是保留的关键字,未来可能作为关键字

标志符

标志符就是给程序中类,接口,方法,变量等命名的符号,
标志符可以由数字,字母,下划线和\$组成,也可以包含关键字和保留字,但是不能是关键字和保留字
标志符不能以数字开头,不能包含空格,只包含下划线和\$符号,其他符号都不能

常量

常量:值不能改变的变量叫常量,常量分为标志符常量和字面常量,标志符常量被初始化后值就不能改变.字面常量就是java的true,false,12等等,存储在内存中,用户无法改变它的值.

变量

变量是内存中的一块存储空间,用来存放变量.
如果数据的值不确定,这时候就可以定义变量,但不是必须.
变量组成三要素,数据类型,变量名,初始化的值
变量的生命周期:
从定义的位置开始生效,到所在{}的}时结束.

数据类型

基本数据类型:byte short char int float long double boolean
引用数据类型:数据,类,接口
数据类型自动提升:低字节类型自动转换成高字节类型,布尔类型不参加转换
反向需要强制转换,可能会丢失精度 .
规则:byte/char/short -> int -> long -> float -> double

++,–,自加,自减

>, \< ,>=, \<= ,== ,=== 比较运算符
&,\&\&,|,||,!,^

springMVC常用注解详解(一)

springMVC常用注解详解(一)

参考

1.springmvc常用注解标签详解:https://www.cnblogs.com/leskang/p/5445698.html

正文

1. @Controller

Spring mvc,model,view,controler,
@controller就是所谓的控制器,DispatcherServlet分发的请求,它将用户请求的数据 通过业务处理层@service转化成为一个model,返回给相应的view展示.
使用@controller,只需要对一个类标注@controller,然后在对应类或者方法上使用@RequestMapping 和@RequestParam 等一些注解用以定义URL 请求和Controller 方法之间的映射，这样controller的方法就能被网络请求访问到.

ps:类标注@controller还不能使用,要在spring配置文件中配置扫描bean路径:
< context:component-scan base-package = “com.ykf.test.web” />

2. @RequestMapping

@RequestMapping 用在controller类的方法或类上面用来处理地址映射的注解.如果用在类上,类上的路径是所有方法的父路径.

requestmapping的属性:

  • values 实际路径
  • method 请求的method类型,比如get,post等等
  • consumes
  • produces
  • params 指request里必须包含的参数,否则无法处理
  • headers 指request的header里必须包含的数据,否则无法处理

3. @autowired 和 @Resource

@autowired 和 @Resource 都是bean的注入使用.
差异共同点请看参考1,总结来说就是@autowired 和 @Resource 都可以用在属性和setter上,但是用在属性上就不需要再写setter方法了.@autowired是根据type实现注入的,@resource有name和type两种属性,哪个属性赋值了就用属性来查找注入,
两个属性都不赋值默认根据name注入.

5. @PathVariable

带占位符的 URL 是 Spring3.0 新增的功能,通过 @PathVariable 可以将 URL 中占位符参数绑定到控制器处理方法的入参中,就是可以把uri里的变量作为传入参数

java 元注解(2)--@Retention

java 元注解(2)--@Retention

参考

  1. annotation(@Retention@Target)详解:https://www.cnblogs.com/gmq-sh/p/4798194.html

解释(下面一段话引用自参考1)

@Retention定义了该Annotation被保留的时间长短：某些Annotation仅出现在源代码中，而被编译器丢弃；而另一些却被编译在class文件中；编译在class文件中的Annotation可能会被虚拟机忽略，而另一些在class被装载时将被读取（请注意并不影响class的执行，因为Annotation与class在使用上是被分离的）。使用这个meta-Annotation可以对 Annotation的“生命周期”限制。

正文

@Retention定义了注解的生命周期(意思就是这个注解在什么范围内生效),java.lang.annotation.RetentionPolicy里的枚举值是它所有的取值范围.

值 注解生效范围
SOURCE 在源文件中有效（即源文件保留）
CLASS 在class文件中有效（即class保留）
RUNTIME 在运行时有效（即运行时保留）
