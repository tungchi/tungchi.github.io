java 元注解(1)--@Target

java 元注解(1)--@Target

解释

元注解(meta-annotation),元注解就是用来注解其他注解的注解.
java1.5引入了4个元注解:@Target,@Retention,@Documented,@Inherited

参考地址:

annotation(@Retention@Target)详解:https://www.cnblogs.com/gmq-sh/p/4798194.html
spring中注解处理框架解析—-源代码实现:https://blog.csdn.net/hao707822882/article/details/39204745
@target:https://blog.csdn.net/u012992462/article/details/80675932

@target

@target是用来设定注解的使用范围的,@target({ElementType.METHOD}).
ElementType是一个枚举类,提供注解可能出现在Java程序中的语法位置（与@Target一起指定在何处写入注解是合法的）

值 注解使用范围
METHOD 用在方法上
TYPE 用在类,接口(包含注解),枚举上
FIELD 用在属性上,包含枚举常量
PARAMETER 用在参数上
CONSTRUCTOR 用在构建方法上
LOCAL_VARIABLE 用在局部变量上
ANNOTATION_TYPE 可用于注解类型上（被@interface修饰的类型）
PACKAGE 用在固定的文件package-info.java

解释

引用网上的一段话:

@interface 是用来修饰 Annotation （注解）的，它不是 interface。这个关键字声明隐含了一个信息：它是继承了java.lang.annotation.Annotation 接口，而不是声明了一个 interface。
Java 从1.5开始提供了 Annotation （注释，标注），它用来修饰应用程序的元素（类，方法，属性，参数，本地变量，包、元数据），编译器将其与元数据一同存储在 class 文件中，运行期间通过 Java 的反射来处理对其修饰元素的访问。Annotation 仅仅用来修饰元素，而不能影响代码的执行。只有通过其配套的框架或工具才能对其信息进行访问和处理。
这段话的重点在于,注解只是注解,我们必须有另外一套方法或者工具或者框架处理对应的注解的元素.
