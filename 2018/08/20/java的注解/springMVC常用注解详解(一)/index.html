springMVC常用注解详解(一)

springMVC常用注解详解(一)

正文

1. @Controller

Spring mvc,model,view,controler,
@controller就是所谓的控制器,DispatcherServlet分发的请求,它将用户请求的数据 通过业务处理层@service转化成为一个model,返回给相应的view展示.
使用@controller,只需要对一个类标注@controller,然后在对应类或者方法上使用@RequestMapping 和@RequestParam 等一些注解用以定义URL 请求和Controller 方法之间的映射，这样controller的方法就能被网络请求访问到.

ps:类标注@controller还不能使用,要在spring配置文件中配置扫描bean路径:
< context:component-scan base-package = “com.ykf.test.web” />

2. @RequestMapping

@RequestMapping 用在controller类的方法或类上面用来处理地址映射的注解.如果用在类上,类上的路径是所有方法的父路径.

requestmapping的属性:

  • values 实际路径
  • method 请求的method类型,比如get,post等等
  • consumes
  • produces
  • params 指request里必须包含的参数,否则无法处理
  • headers 指request的header里必须包含的数据,否则无法处理

3. @autowired 和 @Resource

@autowired 和 @Resource 都是bean的注入使用.
差异共同点请看参考1,总结来说就是@autowired 和 @Resource 都可以用在属性和setter上,但是用在属性上就不需要再写setter方法了.@autowired是根据type实现注入的,@resource有name和type两种属性,哪个属性赋值了就用属性来查找注入,
两个属性都不赋值默认根据name注入.

5. @PathVariable

带占位符的 URL 是 Spring3.0 新增的功能,通过 @PathVariable 可以将 URL 中占位符参数绑定到控制器处理方法的入参中,就是可以把uri里的变量作为传入参数

6. @requestParam

@requestParam就是类似request.getParameter(“”),下面引用自参考1.

它有三个常用参数：defaultValue = “0”, required = false, value = “isApp”；defaultValue 表示设置默认值，required 通过boolean设置是否是必须要传入的参数，value 值表示接受的传入的参数类型。

7、@ResponseBody

@ResponseBody用于将Controller的方法返回的对象,通过适当的HttpMessageConverter转换为指定格式后，写入到Response对象的body数据区。方法返回值不是html页面路径的时候可以用,目前基本上都是将其转化成json.
