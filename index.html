java-threadLocal类的解析应用

java-threadLocal类的解析应用

1. 参考

Java并发编程：深入剖析ThreadLocal:https://www.cnblogs.com/dolphin0520/p/3920407.html

2. 为什么使用threadLocal

1.假设有一个共享变量,在多线程中使用,就会存在问题,我们必须进行同步处理,但是这样效率就会降低.这时候需要分析
这个共享变量有没有必要,如果有必要,那不能使用ThreadLocal,如果没必要,我们就可以使用ThreadLocal.
ThreadLocal会在每个线程中建一个变量,线程之间不影响,但是会加大内存开销.

3. 解析ThreadLocal

我们最常用的方法

get()方法是用来获取ThreadLocal在当前线程中保存的变量副本，set()用来设置当前线程中变量的副本，remove()用来移除当前线程中变量的副本

先看get方法具体实现:

先是获取当前线程t,然后调用getMap()传入t

getMap()返回的是当前线程的一个变量

ThreadLocalMap 是ThreadLocal的内部类

ThreadLocalMap 的entry继承了weakReference(弱引用,后续会讲到),ThreadLocal作为key
接get()以上,如果map为null或者entry为null,返回setInitialValue()

setInitialValue显示获取当前线程的ThreadLocalMap,如果为空,创建,否则set.

4. ThreadLocal应用场景

数据库管理,session管理等,举例代码参考上面的链接.
我曾经在日期格式化里用过
private static final ThreadLocal df = new ThreadLocal() {
@Override
protected SimpleDateFormat initialValue() {
return new SimpleDateFormat(“yyyy-MM-dd”);
}
};

1. web.xml 的第一行

web.xml的第一行定义了该web.xml的版本描述和语言编码方式。

2. web.xml的根节点

  • 官方文档说明:官方文档
  • 参考链接:https://blog.csdn.net/u012453500/article/details/42320451

<web-app><web-app>是web.xml的根节点。xml的schema都必须放在xml文件的根标签上。同理其他类型的xml的schema也都必须放在根标签上。
schema的官方描述是XML Schemas for Java EE Deployment Descriptors,大意是javaee部署描述符,作用大概是用来描述文档结构的.Schema是一个Xml.

引用方式

xmlns：是系统识别的关键字，可以理解为一种变量类型
简称：就是名称空间的简写形式，一个引用，指向定义的名称空间
名称空间：通常名称空间是一个http地址，不过这个URI不指文件，可以看为一个值唯一的字符串标记
schemaLocation：其中定义了约束文档Schema的地址，这个关键字来自于xsi
xsi：定义的变量名，引用的是之后的名称空间，这个名称空间是个著名空间，会被Schema解析引擎所识别，会去找这个命名空间所对应的Schema文档

通过hwclock –systohc将系统时间同步到硬件时间

所有可以通过date -s “YYYY-MM-DD hh:mm:ss” &&hwclock –systohc来修改系统时间

ntpdate同步时间比较好,上面的重启之后不会生效

所以写个定时脚本,然后开机自启脚本

chkconfig –list|grep crond 查看定时任务是否自动运行

crontab -e

然后编辑
*\/10 * * * * ntpdate cn.pool.ntp.org
保存

crontabl -l 可以查看定时任务

/var/spool/mail/root 定时任务执行发送邮件

关于文件上传大小限制的一次问题记录

关于文件上传大小限制的一次问题记录

问题:用公网ip访问上传文件会报403错误.

解决思路:
开始的时候,就从网上查阅资料,修改nginx.conf配置,但是外网还是不能上传文件.
后来想了下用内网默认80端口上传文件,发现可以上传.(这里应该就可以看出应该不是本机服务器的nginx配置原因)
但是因为不自信,还是查阅网上各种方法去修改实验,还是不行.浪费了一天时间.
最后在日报中向领导报告这个问题.
老大认为可能是别的服务器的转发的时候作的限制,后来发现果然是这个问题.
